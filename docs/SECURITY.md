# Security Policy

## Supported Versions

We are committed to maintaining the security of Spacedey. Currently supported versions receive security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.x.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security vulnerability in Spacedey, please follow these steps:

### How to Report

1. **Do not** create a public GitHub issue for security vulnerabilities
2. Email your findings to: **security@spacedey.com**
3. Include the following information:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if available)

### What to Expect

- **Initial Response:** We will acknowledge receipt of your report within 24-48 hours
- **Assessment:** We will investigate and assess the vulnerability within 5-7 business days
- **Updates:** We will provide regular updates on the status of your report every 7 days
- **Resolution:** We will work on a fix and notify you of the timeline
- **Disclosure:** Once patched, we will credit you (if desired) and publish a security advisory

### Vulnerability Outcomes

- **Accepted:** If the vulnerability is confirmed, we will:
  - Create a security patch in the next available release
  - Notify all users of the vulnerability and patch
  - Credit the reporter

- **Declined:** If the report is not a valid vulnerability, we will:
  - Explain why we don't consider it a security issue
  - Provide guidance if applicable

## Security Best Practices

When using Spacedey:

- Keep your dependencies updated
- Use strong passwords and enable two-factor authentication
- Report suspicious activity immediately
- Never share API keys or sensitive credentials

## Contact

For security inquiries, please contact: **security@spacedey.com**
